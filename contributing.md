# Contributing to LineBlox

Thank you for your interest in contributing to **LineBlox**!
By contributing, you help improve the tool for everyone.

---

## Ways to Contribute

1. **Report Bugs**  
   - Use the [Issues](https://github.com/PilotTim136/LineBlox/issues) page to report any bugs or problems.
   - Include steps to reproduce the issue and, if possible, a screenshot or code snippet.

2. **Suggest Features**  
   - Open a new issue and describe the feature you want to see.
   - Explain why it would be useful and how it might work.

3. **Submit Code**  
   - Fork the repository.
   - Create a new branch for your feature or bug fix:
     ```bash
     git checkout -b my-feature
     ```
   - Make your changes.
   - Run the project locally to test your changes.
   - Commit your changes with clear messages:
     ```bash
     git commit -m "Add [feature/fix]: short description"
     ```
   - Push to your fork and open a Pull Request.

4. **Improve Documentation**  
   - Update README, add examples, or improve explanations.
   - Documentation improvements are always welcome!

---

## Code Style Guidelines

- Use **consistent indentation** (4 spaces (or TAB) recommended for JS/HTML).
- Keep your code **modular** and easy to read.
- Write **descriptive variable and function names**.
- Include **comments** for complex logic.
